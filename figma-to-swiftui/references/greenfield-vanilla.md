# Greenfield-vanilla scaffold (P1-5)

When `mode-detect.sh` classifies the project as `greenfield-vanilla` (or `greenfield-ikame` overridden to vanilla by user choice), use the `scripts/vanilla-scaffold.sh` script to produce a minimal SwiftUI iOS project that's immediately compatible with the figma-to-swiftui workflow and hooks.

This document explains: when to use it, what it produces, what to do after.

---

## When greenfield-vanilla is the right choice

‚úÖ **Choose vanilla when:**
- Project has no Ikame umbrella dependency requirement.
- User explicitly opts out of Ikame (e.g., they're building a personal app, an OSS sample, or a non-Ikame company project).
- `ikxcodegen` is unavailable on the machine.
- The team's Swift Package Manager / xcodegen workflow is preferred over CocoaPods.

‚ùå **Choose `ikxcodegen-scaffold.sh` (Ikame) when:**
- User works at Ikame and the project will use IKCoreApp / IKNavigation / IKMacros.
- The team has shared assets/conventions in the Ikame Pod ecosystem.
- The `ikxcodegen` binary is available.

If unsure, ask the user before scaffolding.

## Usage

```bash
# Minimal:
~/.claude/skills/figma-to-swiftui/scripts/vanilla-scaffold.sh /path/to/MyApp

# With explicit name / bundle / iOS min:
~/.claude/skills/figma-to-swiftui/scripts/vanilla-scaffold.sh /path/to/MyApp \
  --name MyApp \
  --bundle-id com.example.myapp \
  --ios-min 17.0 \
  --mode scaffold

# Skip scaffold mode (start strict):
~/.claude/skills/figma-to-swiftui/scripts/vanilla-scaffold.sh /path/to/MyApp \
  --mode production
```

Refuses to overwrite an existing `project.yml` or `.xcodeproj`. Delete the folder manually if you want to re-scaffold.

## After scaffolding ‚Äî recommended next steps

1. **Pin the Figma file.** Note the `fileKey` from the Figma URL. Add to `c1-conventions.json`:
   ```json
   "figmaFileKey": "qKOTZUKyYFV4GCn4FMMehS",
   "figmaRootNodeId": "3:1812"
   ```

2. **Run `figma_build_registry`.** Save the result to `.figma-cache/_shared/registry.json`. If `screens: []` and `candidateScreens: [...]`, see `references/registry-empty-fallback.md`.

3. **Iterate per feature area.** For each area:
   - Subagent (see `parallel-subagent-pattern.md`) OR inline workflow.
   - Phase A ‚Üí asset export ‚Üí Phase B ‚Üí C5 per screen.

4. **Asset catalog pinning.** Use `<project>/<ProjectName>/Resources/Assets.xcassets` as the canonical `assetCatalogPath` for every `figma_export_assets_unified` call. The scaffold writes the empty catalog with `AppIcon` + `AccentColor` + `AppBackground` already provisioned.

5. **DesignSystem tokens.** Run `b0a-extract-copy.sh` (per screen, accumulating in `Strings.swift`) and `b0b-tokens-codegen.sh` (once per flow, appending to `AppColor.swift` + `AppFont.swift`).

6. **Switch to production mode.** Once Phase A is complete for ALL screens AND `xcodebuild build` succeeds AND C5 PASS per screen, edit `c1-conventions.json` ‚Üí `mode: "production"`. Re-run the full session-end sweep. Fix any blocker that surfaced.

## Comparison to `ikxcodegen-scaffold.sh`

| Aspect | vanilla-scaffold | ikxcodegen-scaffold |
|---|---|---|
| Dependency manager | None (xcodegen + SwiftPM if needed) | CocoaPods (IKCoreApp pod) |
| Routing | Vanilla `NavigationStack` | `IKNavigation` + `MainRouter` |
| State | `ObservableObject` / `@Observable` | `@APIProtocol` + IKMacros |
| Popups | `.sheet` / `.alert` | `IKPopup.shared.showPopup(...)` |
| Tracking | None (add per project need) | `AppTrackingFeature.shared` + IKTracking |
| Asset symbols | `Image(.assetName)` (iOS 17+ auto-gen) | `Image(.assetName)` (auto-gen OR IKAssetSymbol macro) |
| xcconfig wiring | Not needed | `firebase-debug.xcconfig` + `appstore.xcconfig` wired by wrapper |
| Time to first build | ~5s (xcodegen generate) | ~60‚Äì120s (pod install) |

Vanilla is faster to scaffold and review; Ikame ships richer baseline functionality. Pick based on team / target ecosystem, not on speed.

## Caveats

- **No CocoaPods support out of the box.** If you need Pods later, add a Podfile manually and run `pod init`. The vanilla scaffold assumes SwiftPM-only or no third-party deps.
- **No localization scaffolding.** `Strings.swift` is plain Swift enums; if you need `LocalizedStringKey` flow, see `references/iklocalized-bridge.md` and adapt for vanilla (use `String(localized:)`).
- **No Lottie scaffolding.** If the design has eAnim* placeholders, install Lottie via SwiftPM and replace `LottieView` placeholder per `lottie-placeholders.md`.
- **No analytics/telemetry.** Add per project need. The scaffold doesn't presume Firebase/Mixpanel/etc.
